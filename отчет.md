# Отчет

## Реализованные технологии

### 1. Pub/Sub + Множественные подписчики

**Файлы:**

- `broker.py`: Реализован класс `MessageBroker`, который управляет топиками и подписками. Подписчики добавляются через метод `add_subscriber`.
- `topic.py`: Содержит класс `Topic`, который обрабатывает публикации и распространение сообщений для всех подписчиков.

**Описание:**
Система поддерживает подписчиков на один и тот же топик, где каждый подписчик получает копию каждого сообщения, что реализует паттерн pub/sub с множественными подписчиками.

### 2. Персистентность + Восстановление + At-least-once Delivery

**Файлы:**

- `storage.py`: Реализует класс `JsonlLogStorage` для хранения сообщений, оффсетов и метаданных в файловой системе.
- `broker.py`: Управляет сохранением и восстановлением состояния из файловой системы.

**Описание:**
Сообщения и состояния сохраняются в JSONL формате с использованием fsync для обеспечения персистентности. При перезапуске брокера состояние восстанавливается, и недоставленные сообщения повторно отправляются подписчикам.

### 3. Python SDK (Publisher + Subscriber)

**Файлы:**

- `publisher.py`: Реализует класс `BrokerPublisher`, предоставляющий методы для публикации сообщений.
- `subscriber.py`: Реализует класс `BrokerSubscriber`, предоставляющий методы для подписки и обработки сообщений.

**Описание:**
Предоставлены клиентские библиотеки для упрощения интеграции с брокером, включая обработку повторных попыток и подтверждений доставки (ACK).

### 4. TTL, DLQ, Приоритеты, Метрики

**Файлы:**

- `topic.py`: Обрабатывает TTL сообщений и помещает их в DLQ при истечении времени.
- `dead_letter_queue.py`: Реализует функциональность DLQ для обработки недоставленных сообщений.
- `metrics.py`: Содержит класс `Metrics` для сбора и предоставления метрик по сообщениям и производительности.
- `enhanced_demo.py`: Демонстрирует работу с приоритетами, TTL и DLQ.

**Описание:**
Реализована поддержка времени жизни сообщений (TTL), Dead Letter Queue (DLQ) для сообщений, которые не удалось доставить, а также система метрик для мониторинга брокера. Поддерживаются приоритеты сообщений для управления порядком их доставки.

### 5. Docker / Docker-compose

**Файлы:**

- `Dockerfile`: Определяет образ для контейнера приложения.
- `docker-compose.yml`: Описывает многоконтейнерную среду с брокером и тестовыми микросервисами.

**Описание:**
Контейнеризация приложения с использованием Docker и Docker-compose позволяет легко разворачивать брокер и связанные микросервисы в изолированной среде.

## Дополнительный функционал

### Producer Deduplication

**Файлы:**

- `broker.py`, `storage.py`: Реализована дедупликация сообщений на стороне продюсера, чтобы избежать повторной публикации одного и того же сообщения, если оно было отправлено повторно.

**Описание:**
Используется кэш дедупликации, чтобы гарантировать идемпотентность публикации сообщений с тем же идентификатором клиентского сообщения.

### WebSocket Push Notifications

**Файлы:**

- `api.py`, `subscriber.py`: Поддержка WebSocket для push-уведомлений, позволяющая подписчикам получать сообщения в реальном времени без необходимости опрашивать сервер.
